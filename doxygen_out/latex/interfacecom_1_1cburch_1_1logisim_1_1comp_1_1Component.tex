{\footnotesize\ttfamily Object com.\+cburch.\+logisim.\+comp.\+Component.\+get\+Feature (\begin{DoxyParamCaption}\item[{Object}]{key }\end{DoxyParamCaption})}

Retrieves information about a special-\/purpose feature for this component. This technique allows future Logisim versions to add new features for components without requiring changes to existing components. It also removes the necessity for the Component API to directly declare methods for each individual feature. In most cases, the {\ttfamily key} is a {\ttfamily Class } object corresponding to an interface, and the method should return an implementation of that interface if it supports the feature.

As of this writing, possible values for {\ttfamily key} include\+: {\ttfamily Pokable.\+class} , {\ttfamily Custom\+Handles.\+class}, {\ttfamily Wire\+Repair.\+class}, {\ttfamily Text\+Editable.\+class }, {\ttfamily Menu\+Extender.\+class}, {\ttfamily Tool\+Tip\+Maker.\+class}, {\ttfamily  Expression\+Computer.\+class}, and {\ttfamily Loggable.\+class}.
