\doxysubsection{Detailed Description}
Represents a category of components that appear in a circuit. This class and {\ttfamily Component } share the same sort of relationship as the relation between {\itshape classes} and {\itshape instances} in Java. Normally, there is only one Component\+Factory created for any particular category.

{\footnotesize\ttfamily Object com.\+cburch.\+logisim.\+comp.\+Component\+Factory.\+get\+Feature (\begin{DoxyParamCaption}\item[{Object}]{key,  }\item[{Attribute\+Set}]{attrs }\end{DoxyParamCaption})}

Retrieves special-\/purpose features for this factory. This technique allows for future Logisim versions to add new features for components without requiring changes to existing components. It also removes the necessity for the Component API to directly declare methods for each individual feature. In most cases, the {\ttfamily key} is a {\ttfamily Class} object corresponding to an interface, and the method should return an implementation of that interface if it supports the feature.

As of this writing, possible values for {\ttfamily key} include\+: {\ttfamily TOOL\+\_\+\+TIP} (return a {\ttfamily String}) and {\ttfamily SHOULD\+\_\+\+SNAP} (return a {\ttfamily Boolean}).
